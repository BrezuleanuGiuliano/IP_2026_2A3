1. DESCRIEREA PROIECTULUI
================================================================================

EduConnect este o platforma educationala care reuneste functionalitati de
invatare online si gasire a unui meditator potrivit. Platforma permite
profesorilor sa publice cursuri structurate (lectii video, documente, text),
sa creeze quiz-uri pentru evaluarea studentilor, si sa ofere meditatii
individuale. Studentii pot parcurge cursurile, sustine evaluari, posta
comentarii pe lectii, si rezerva sesiuni de meditatii cu profesori disponibili.
Administratorii asigura moderarea continutului si gestionarea utilizatorilor.

Actorii sistemului: Student, Profesor, Administrator, Stripe API (extern),
                    Google OAuth (extern), Email Service (extern)

3.1 Diagrame C4 Model (fara AI) - Issue #1
--------------------------------------------------------------------------------

Am realizat cele 3 niveluri ale modelului C4 pentru platforma EduConnect:

  C1 - Context Diagram:
    - Sistemul central: EduConnect Platform
    - Persoane: Student, Profesor, Administrator
    - Sisteme externe: Stripe API (plati), Google OAuth (autentificare),
      Email Service (notificari)
    - Relatii cu protocoale: HTTPS, OAuth 2.0, SMTP

  C2 - Container Diagram (zoom in pe EduConnect Platform):
    - Web Application (React) - interfata web
    - Mobile Application (React Native) - aplicatie mobila cross-platform
    - Backend API (Spring Boot / Java) - logica de business, REST API
    - Database (PostgreSQL) - stocarea tuturor datelor
    - Notification Service (Java / Spring) - notificari in-app, email, push
    - Protocoale: JSON/HTTPS, JDBC/SQL, Async/REST, SMTP, OAuth 2.0

  C3 - Component Diagram (zoom in pe Backend API):
    - Authentication Component (Spring Security) - register, login, OAuth, JWT
    - Course Management Component - CRUD cursuri, adaugare lectii
    - Quiz Engine Component - quiz-uri, intrebari, anti-cheat, corectare automata
    - Comment System Component - comentarii pe lectii, raspunsuri, best answer
    - Marketplace Component - profiluri tutori, cautare, rezervare sesiuni
    - Payment Component (Stripe SDK) - procesare plati, refund
    - Admin Component - gestionare utilizatori, moderare continut
    - Enrollment & Progress Component - inscriere cursuri, urmarire progres

  Tool utilizat: diagrams.net cu plugin-ul C4 Draw.io

3.2 Clase Java fara AI - Issue #2
--------------------------------------------------------------------------------

  Course.java:
    - Atribute: id (UUID), title, description, category, difficulty,
      status (CourseStatus), createdAt, lessonIDs, enrollmentIDs
    - Metode importante:
      * publish() - valideaza ca cursul e in starea DRAFT si are cel putin
        o lectie inainte de publicare
      * archive() - verifica ca nu e deja arhivat
      * addLesson() - adauga lectie cu verificare de duplicate
    - Enum intern: CourseStatus (DRAFT, PUBLISHED, ARCHIVED)
    - Utilizeaza Lombok (@Getter, @Setter, @AllArgsConstructor)

  Enrollment.java:
    - Atribute: id (UUID), userID, courseID, progressPercent, enrolledAt,
      commentIDs, totalLessonsCount, solvedLessonsCount
    - Metode importante:
      * enroll() - confirma inscrierea
      * updateProgress() - incrementeaza lectiile rezolvate, recalculeaza
        procentul, anunta completarea cursului
      * calculateProgress() - formula: (solvedLessons / totalLessons) * 100,
        trateaza impartirea la zero
    - Utilizeaza Lombok (@Getter, @Setter, @NoArgsConstructor)

3.3 Clase Java cu AI - Issue #3
--------------------------------------------------------------------------------

  User.java:
    - Atribute: id (UUID), email, passwordHash, name, avatarUrl,
      role (UserRole), emailVerified, enrollments, quizAttempts,
      bookings, comments, createdAt
    - Metode importante:
      * register(email, password, role) - factory method static, valideaza
        email si parola, hash-uieste parola, extrage numele din email
      * login(email, password) - valideaza credentialele si verificarea
        emailului, returneaza token JWT-like
      * loginWithOAuth(provider) - autentificare OAuth, returneaza token
        specific providerului
      * resetPassword(email) - genereaza token de resetare
      * verifyEmail(token) - marcheaza emailul ca verificat
    - Enum intern: UserRole (STUDENT, TEACHER, ADMIN)

  AutoGrader.java:
    - Componenta Spring @Service (stateless)
    - Metode importante:
      * grade(QuizAttempt) - numara raspunsurile corecte, calculeaza scorul
        brut, aplica penalizare de -2% per tab-switch suplimentar (anti-cheat),
        calculeaza scorul final, afiseaza statistici detaliate
      * getFeedback(score, passingScore) - returneaza mesaj promovat/nepromovat
      * getStatistics(QuizAttempt) - returneaza obiect QuizStatistics
    - Clasa interna: QuizStatistics (score, passed, timeTakenSeconds, tabSwitches)

4.3 Diferente intre codul scris manual vs cu AI
--------------------------------------------------------------------------------

  Fara AI (Course.java, Enrollment.java):
  - Cod mai concis, foloseste Lombok pentru boilerplate
  - Logica de business focusata pe operatii CRUD simple
  - Validari de baza (status check, duplicate check, division by zero)

  Cu AI (User.java, AutoGrader.java):
  - Cod mai detaliat si mai verbose (getters/setters expliciti in User)
  - Logica de business mai complexa (flux autentificare complet,
    sistem de scoring cu penalizari)
  - Output de logging mai detaliat cu formatare
  - Clase interne suplimentare (QuizStatistics)
  - Tratare exceptii mai granulara (SecurityException, IllegalArgumentException)
